限制条件：
每个lane的数据量，碱基平衡，汉明距离，不平衡文库，加急，F（单端文库可以同双端文库序列汉明距离为0）

1，
读取excel数据，
每行是一个子文库对象（Library）
相同吉因加编号的子文库构成一个文库组对象（LibraryGroup）
如果是备注中有“同lane上机”，则项目名称相同，且吉因加编号相同的放到一个文库组中

2，
计算总数据量，按每个lane的数据量，规划出需要几个lane
洗脱文库按数据量大小排序
以洗脱文库为单位，逐个放入每个lane中，新洗脱文库数据放到数据量小的lane中，保持多个lane数据平衡
（排序的原因：避免最后一个洗脱文库放不到lane中，
例如：一个lane的总量是1400，剩下一个80，第一个lane1330，第二个lane1340，总的余量是130，但是两个lane都放不下这个80
如果排序后还是放不下最后一个，那就直接放到未排单洗脱文库里面）
组合记录下来

3，
放好洗脱文库之后，开始计算汉明距离，一个lane中如果汉明距离不合要求，先将汉明距离冲突的分到不同的lane中，如果冲突的洗脱文库数量大于总lane数，
将数据量小的洗脱文库放到未排单中，再次按数据量进行排单
重复第三步，直至汉明距离符合要求
组合记录下来

4，
按排单，计算碱基比例，
a，如果碱基平衡不符合要求，尝试将多个lane中数据量最大的洗脱文库进行交换，通过之前的组合记录，排除汉明距离不符合条件的情况
b，如果碱基平衡不符合要求，找出碱基平衡不满足条件的碱基比率，然后查找其他lane中洗脱文库中对应碱基数据量大的，同本lane中洗脱文库中对应碱基数据量小的进行交换
再次计算汉明距离，直至汉明距离符合要求，再次计算碱基平衡
重复第四步，直至碱基平衡符合要求
组合记录下来

5，
一种暴力算法，将所有洗脱文库按数据量要求进行排列组合，然后将所有排列组合进行汉明距离和碱基平衡的计算，打分，找到最合适的

6，
在将1个洗脱文库加入到lane中时计算汉明距离，碱基平衡，如果不符合条件，加到另外的lane中，如果都不符合，就加2个洗脱文库，依次类推
有一个bug，如果第一个文库数据量很大，且不平衡，有可能整个排单都不行，意思是去掉这个洗脱文库之后其他的就可以了，这样的情况，上面的做法处理不了

动态规划：将洗脱文库再分组，每组都要满足碱基平衡的条件，多组合并在一起必然也满足碱基平衡的条件
